# Screenshot Guide for MyFinPal App Store Submission

## Required Screenshot Sizes

### iPhone Screenshots (Required)
1. **iPhone 6.7"** (iPhone 14 Pro Max, 15 Pro Max)
   - Size: 1290 x 2796 pixels
   - Required: Yes
   - Minimum: 3 screenshots, Maximum: 10

2. **iPhone 6.5"** (iPhone 11 Pro Max, XS Max)
   - Size: 1242 x 2688 pixels
   - Required: Yes
   - Minimum: 3 screenshots, Maximum: 10

3. **iPhone 5.5"** (iPhone 8 Plus)
   - Size: 1242 x 2208 pixels
   - Required: Yes
   - Minimum: 3 screenshots, Maximum: 10

### iPad Screenshots (If supporting tablets)
4. **iPad Pro 12.9"**
   - Size: 2048 x 2732 pixels
   - Required: Only if `supportsTablet: true` (which you have)
   - Minimum: 3 screenshots, Maximum: 10

## How to Capture Screenshots

### Option 1: iOS Simulator (Easiest)
1. Open your app in iOS Simulator
2. Navigate to the screen you want
3. Use device menu: **Device > Screenshots > [Device Name]**
4. Screenshots saved to Desktop
5. Resize if needed using image editing software

### Option 2: Physical Device
1. Take screenshots on your iPhone/iPad
2. Transfer to computer
3. Resize to required dimensions using image editing software

### Option 3: Design Tools (Recommended for Professional Look)
1. Use Figma, Sketch, or Photoshop
2. Create mockups with your app screens
3. Add text overlays and annotations
4. Export at exact required sizes

## Screenshot Best Practices

### Do's ✅
- Use real data (but anonymized/sample data)
- Show the app's best features
- Keep UI clean and uncluttered
- Use consistent styling across screenshots
- Add text overlays to highlight key features (optional but recommended)
- Show the app in use (not just static screens)
- Use your brand colors consistently

### Don'ts ❌
- Don't use placeholder text like "Lorem ipsum"
- Don't show error states or empty states (unless highlighting a feature)
- Don't include device frames (Apple adds them)
- Don't add rounded corners (Apple adds them)
- Don't include status bar (unless it's part of your design)
- Don't use copyrighted images or content
- Don't show personal/real financial data

## Quick Workflow

1. **Capture screenshots** from simulator or device
2. **Resize** to exact required dimensions
3. **Add text overlays** (optional but recommended)
4. **Review** for quality and clarity
5. **Export** as PNG files
6. **Upload** to App Store Connect

## Example Screenshot Checklist

For each device size, prepare:
- [ ] Screenshot 1: Onboarding/Welcome
- [ ] Screenshot 2: Net Worth Dashboard

- [ ] Screenshot 4: Category Breakdown

**Total:** 15 screenshots minimum (3 device sizes × 5 screenshots each)

## Tips

1. **Start with iPhone 6.7"** - This is the most common size, and you can resize down
2. **Use consistent data** - Use the same sample data across all screenshots for consistency
3. **Test on device** - Make sure screenshots look good when viewed on an actual iPhone
4. **Update regularly** - You can update screenshots without a new submission
5. **A/B test** - Try different screenshot orders to see what converts best
